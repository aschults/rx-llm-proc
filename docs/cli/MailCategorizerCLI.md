# `mail_categorizer_cli`: Email Categorization Tool

## Introduction

The `mail_categorizer_cli` tool uses the Gemini API to automatically categorize
emails based on their content. It works with an `index.json` file generated by
`gmail_cli get_all --with_index`.

## Authentication

This tool requires a Gemini API key to communicate with the Gemini API. It is recommended to provide this through the `GEMINI_API_KEY` environment variable:

```bash
export GEMINI_API_KEY="your_api_key_here"
```

---

## `categorize` Command


Processes an email index and assigns categories to each email.

### Usage

```shell
mail_categorizer_cli categorize [OPTIONS]
```

### Arguments and Options

| Argument/Option                   | Description                                                                                             |
| :-------------------------------- | :------------------------------------------------------------------------------------------------------ |
| `--index_file <PATH>`             | **(Required)** Path to the source `index.json` file from `gmail_cli`.                                   |
| `--categorized_index_file <PATH>` | **(Required)** Path to the categorized index file for reading existing and saving new results.          |
| `--emails_dir <PATH>`             | **(Required)** Path to the root directory where email content files are stored (used for refinement).   |
| `--prompt <PATH>`                 | **(Required)** The prompt template file (Jinja2) for the initial categorization.                        |
| `--refine_prompt <PATH>`          | The prompt template file for the detailed refinement step.                                              |
| `--parameter` / `-P <KEY=VALUE>`  | A key=value parameter to expand in the prompt templates. Use `key=@file` to read the value from a file. |
| `--batch_size <NUM>`              | Number of emails to process per categorization API call.                                                |
| `--model <NAME>`                  | The Gemini model for the initial categorization step.                                                   |
| `--refine_categories <CAT...>`    | A list of categories that should be sent for a second, more detailed refinement analysis.               |
| `--refine_model <NAME>`           | The Gemini model to use for the refinement step. Defaults to the main model.                            |
| `--save_interval <NUM>`           | How often to save the categorized index file (in number of new entries).                                |

### Example

Categorize unread emails using a specific prompt template.

```bash
mail_categorizer_cli categorize \
  --index_file "./unread_emails/index.json" \
  --categorized_index_file "./unread_emails/categorized_index.json" \
  --emails_dir "./unread_emails" \
  --prompt "categorization_prompt.j2"
```

**Sample `categorized_index.json` entry:**

```json
{
  "18d6b1d034b7f8b9": {
    "category": "Project Update",
    "priority": "High",
    "summary": "Status update on the Q1 roadmap.",
    "action_items": ["Review the new design mockups by Friday."]
  }
}
```
